## v1.7.1 Recognition + Telemetry Fix Plan

### Scope
- Fix splash screen version text so it always reflects `BuildConfig.VERSION_NAME`.
- Stabilize release/version policy around semantic versioning:
  - `major` for architecture pivots
  - `minor` for meaningful product behavior changes
  - `patch` for bugfix-only releases
- Fix telemetry auth/config so local release builds can migrate the configured API key and drain staged uploads.
- Suppress speculative costume/event remaps that are causing normal species to resolve as event or costume variants.

### Evidence
- Splash layout still contains a hardcoded version literal.
- Telemetry endpoint probes reach the configured server but runtime config drops the API key, resulting in `403`.
- Diagnostics show same-species authoritative remaps and live-event rescue paths are too permissive for weak non-base classifier signals.

### Implementation
1. Add regression coverage for:
   - weak same-species costume/event rescue
   - generic variant metadata exposure
   - speculative same-species remaps without date-backed support
2. Bind splash version text dynamically from `BuildConfig.VERSION_NAME`.
3. Add `SCAN_TELEMETRY_API_KEY` build config support for local/env-driven builds and migrate it into encrypted telemetry prefs when missing.
4. Skip unauthenticated telemetry probes when an API key is configured and let authenticated uploads determine status.
5. Tighten same-species/event remap thresholds and remove generic-derived metadata exposure for explanation surfaces.
6. Bump to `1.7.1`, verify, build, install, launch, publish.

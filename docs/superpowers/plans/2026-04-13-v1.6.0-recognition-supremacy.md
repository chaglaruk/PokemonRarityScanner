# V1.6.0 Recognition Supremacy Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Eliminate current telemetry/release infrastructure glitches, keep cold start idle, improve recognition gating and live event context, and ship a verified `v1.6.0` release APK.

**Architecture:** Keep the existing scan pipeline intact and target only proven weak points from live logs: telemetry upload semantics, startup state reset, diagnostics completeness, classifier noise suppression, and event context ingestion through the existing event database. Avoid broad refactors.

**Tech Stack:** Kotlin, Room, Coroutines, Jetpack Compose, existing HttpURLConnection clients, GitHub Releases API, current OCR/vision stack.

# v1.7.0 Evolution Plan

## Goal
- Pivot from hybrid OCR + IV solving to a recognition-first scanner.
- Remove IV, appraisal, arc, and power-up-cost dependencies from the live product path.
- Replace Tesseract-first OCR with ML Kit-first recognition and dynamic name-region discovery.
- Keep live event context and telemetry online while simplifying the result surface.

## Evidence Driving The Pivot
- Recent diagnostics still show `stardust` frequently null while species recognition remains usable.
- HP and power-up recovery dominate latency without improving the primary product outcome.
- Latest exact/range outputs are not the product goal anymore; the product now prioritizes fast recognition, rarity, form/costume, and event context.

## Execution Tracks

### 1. Great Purge
- Remove `IvCostSolver`, `ArcPointAnalyzer`, `AppraisalBarAnalyzer`, PvP summaries, and power-up OCR from production flow.
- Remove IV-related model fields from result, telemetry, diagnostics, and UI.
- Remove Tesseract dependency and `tessdata/eng.traineddata`.

### 2. ML Kit Blaze
- Rebuild `OCRProcessor` around ML Kit.
- Add dynamic name ROI discovery using `Text.TextBlock` geometry from a top-half pass.
- Keep CP/HP extraction lightweight and region-targeted with ML Kit only.

### 3. Living Database
- Add a remote metadata sync layer that can refresh species/variant/event metadata on startup.
- Persist the latest snapshot locally and merge it into repository lookups without requiring an APK refresh.

### 4. Recognition & Rarity
- Keep classifier + visual detector as the main recognition stack.
- Tighten variant gating so weak classifier outputs cannot manufacture shiny/costume/form.
- Reweight rarity around species rarity, visual rarity, and live event boosts.

### 5. Infrastructure
- Ensure startup remains clean/idle with no ghost result.
- Keep release upload stable via workflow + local publisher retry path.
- Update README files to reflect the recognition-first architecture.

## Verification
- Focused unit tests for:
  - ML Kit OCR parsing helpers
  - telemetry payload/repository changes
  - variant authority/gating
  - overlay idle state
  - event context refresh and remote metadata parsing
- `:app:assembleRelease`
- install and launch on the connected device
- publish `v1.7.0` release asset

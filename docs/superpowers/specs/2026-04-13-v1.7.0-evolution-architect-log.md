# V1.7.0 Evolution Architect Log

## ROI strategy

- Primary dynamic signal: ML Kit text blocks across the upper 58% of the card.
- Selection rule: prefer centered blocks with the strongest `TextParser.rankNameCandidates(...)` score.
- Fallback: fixed `REGION_NAME` crop with ML Kit text recognition.
- Numeric OCR remains targeted:
  - `REGION_CP` for CP
  - `REGION_HP`, `REGION_HP_ALT`, `REGION_HP_LOWER` for HP / max HP

This removes the previous Tesseract dependency and the power-up-row dependency from the active recognition path.

## Living DB sync

- `RemoteMetadataSyncManager` fetches `metadata_manifest.json` from GitHub RAW.
- Manifest points to:
  - `rarity_manifest.json`
  - `variant_catalog.json`
  - `authoritative_variant_db.json`
- Downloaded files are written under `files/remote_metadata/`.
- `RarityUpdater.syncAsync()` refreshes remote metadata first, resets in-memory loaders, then refreshes live event context.
- `ScanTelemetryPayload.debug.livingDbVersion` reports the active synced version.

## Visual detection methods

- OCR:
  - ML Kit text recognition only
  - OpenCV-backed preprocessing via `ImagePreprocessor`
- Variant resolution:
  - existing species/variant/shiny matcher stack remains active
  - final result is recognition-first, with event context and badges surfaced ahead of raw stats
- Diagnostics:
  - recognition diagnostics remain exportable through `OcrDiagnosticsExporter`
  - telemetry now carries species/form/event/latency rather than IV payload fields
